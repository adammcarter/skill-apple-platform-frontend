# iOS And iPadOS Notes

Use this reference for iPhone and iPad frontend work.

## Core Expectations

- Respect safe areas, touch ergonomics, and predictable navigation patterns.
- Keep the primary action clear and reachable.
- Use sheets, popovers, tab bars, split views, and toolbars according to task complexity and device size.

## Layout

- Avoid desktop-style density on iPhone.
- Let content breathe, but do not waste space on empty framing.
- Make text scaling and localization resilience part of the first implementation.

## Interaction

- Touch targets should feel comfortable, not merely technically large enough.
- Gestures should have visible affordances or discoverable alternatives.
- iPad can support richer keyboard and pointer interaction than iPhone; use that when it helps.
