# macOS Notes

Use this reference for macOS frontend work.

## Core Expectations

- Design for pointer precision and keyboard-first use.
- Support resizable windows and changing layout width gracefully.
- Use menus, context menus, toolbars, sidebars, inspectors, and multi-pane layouts where they fit the task.

## Density And Layout

- macOS can support denser information than iOS, but density should improve throughput, not create clutter.
- Prefer strong alignment and clear grouping.
- Sidebars and inspectors should feel stable and information-rich.

## Interaction

- Hover and focus states matter.
- Secondary click behavior is often worth supporting.
- Keyboard shortcuts and command discoverability are part of product quality, not optional extras.

## Visual Style

- Favor restraint over large mobile-style cards and oversized controls.
- Use materials and ornamentation sparingly so content remains primary.
